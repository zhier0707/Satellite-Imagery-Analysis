> **Web端可视化项目完整方案（含高德开放平台 Skill 集成）**
> 
> 核心模型：EfficientNetV2-M  
> 应用领域：卫星遥感图像分类与土地利用监测  
> 地图服务：高德开放平台 Skill

---

## 1. 项目概述

### 1.1 项目背景

卫星遥感技术近年来发展迅速，已广泛应用于土地利用监测、城市规划、农业估产、环境评估等领域。传统的卫星图像分析方法依赖人工判读，效率低且成本高。随着深度学习技术的成熟，基于卷积神经网络（CNN）的自动化解译成为研究热点。

EfficientNetV2-M 是 Google 于 2021 年提出的新一代高效卷积网络，相比初代 EfficientNet 系列，具有**训练速度快、参数效率高**的显著优势。本项目选择 EfficientNetV2-M 作为核心模型，并集成**高德开放平台 Skill** 提供地理空间可视化能力，旨在构建一个功能完善、用户体验良好的卫星图像分析 Web 平台。

### 1.2 项目目标

- [ ] 实现卫星图像的自动化土地利用类型分类（住宅区、商业区、工业区、农田、森林、水体等）
- [ ] 提供直观的 Web 可视化界面，支持图片上传、实时分类、结果展示
- [ ] 展示分类置信度热力图，增强模型可解释性
- [ ] 支持区域变化检测对比分析
- [ ] 导出分类统计报表，支持城市规划与决策
- [ ] **集成高德地图底图与热力图可视化，实现地理空间叠加展示**
- [ ] **支持地理编码、POI 搜索等 LBS 服务，增强分析能力**

### 1.3 技术选型理由

**EfficientNetV2-M vs ResNet-50 对比：**

| 对比项 | EfficientNetV2-M | ResNet-50 |
|--------|-----------------|-----------|
| 参数量 | 54M | 25M |
| ImageNet Top-1 | **85.0%** | 76.1% |
| 训练速度 | **比 ResNet 快 5 倍** | 基准 |
| 卫星图像适配性 | **高**（渐进式学习+高分辨率） | 中 |

**选择 EfficientNetV2-M 的理由：**
1. **训练速度快**：采用 Fused-MBConv 模块和渐进式学习策略
2. **精度高**：ImageNet Top-1 达 85.0%
3. **参数效率优**：同等精度下参数量更少
4. **支持高分辨率**：适合卫星图像的细粒度分类

## 3. 模型训练与优化

### 3.1 模型架构

EfficientNetV2-M 基于神经网络架构搜索（NAS）设计，采用 Fused-MBConv 模块替代部分深度可分离卷积，显著提升训练和推理效率。

**网络结构特点：**
- 8 个阶段，每个阶段由多个 MBConv 或 Fused-MBConv 模块堆叠
- 全局平均池化 + 全连接层输出分类结果
- 渐进式学习策略：训练初期用小分辨率 → 逐步增大 + 正则化调整

### 3.2 训练配置

| 参数 | 配置值 |
|------|--------|
| **输入尺寸** | 384×384（或根据数据集调整） |
| **Batch Size** | 16-32（根据 GPU 显存调整） |
| **优化器** | AdamW（lr=1e-4，weight_decay=1e-5） |
| **学习率调度** | Cosine Annealing（warmup=5 epochs） |
| **训练轮次** | 50-100 epochs（早停策略） |
| **损失函数** | Label Smoothing Cross Entropy（smoothing=0.1） |
| **混合精度** | FP16 Mixed Precision（节省 40% 显存） |

### 3.3 迁移学习策略

```python
# 三阶段微调策略
Stage 1: 冻结 Backbone，仅训练分类头 (1-2 epochs)
Stage 2: 解冻最后 2-3 个阶段 (10-20 epochs)
Stage 3: 全量微调，使用较小学习率 1e-5
```

### 3.4 模型评估指标

- **总体准确率（Accuracy）**：正确分类样本占总样本比例
- **混淆矩阵**：可视化各类别的分类错误分布
- **F1-Score**：综合考虑精确率和召回率的调和平均值
- **推理速度**：单张图像平均推理时间（ms）

## 6. 核心代码实现

### 6.1 模型训练代码（PyTorch）

```python
import torch
import timm
from torch.utils.data import DataLoader
from torchvision import transforms

# 创建模型
model = timm.create_model("tf_efficientnetv2_m", pretrained=True, num_classes=10)

# 冻结 backbone
for param in model.parameters():
    param.requires_grad = False

# 解冻最后几层
for param in model.classifier.parameters():
    param.requires_grad = True

# 优化器配置
optimizer = torch.optim.AdamW(model.parameters(), lr=1e-4, weight_decay=1e-5)
scheduler = torch.optim.lr_scheduler.CosineAnnealingLR(optimizer, T_max=50)

# 训练循环
for epoch in range(num_epochs):
    model.train()
    for batch in dataloader:
        optimizer.zero_grad()
        outputs = model(batch["image"])
        loss = torch.nn.functional.cross_entropy(outputs, batch["label"])
        loss.backward()
        optimizer.step()
    scheduler.step()
```

### 6.2 Grad-CAM 热力图生成代码

```python
import torch
import cv2
import numpy as np

def generate_gradcam(model, input_tensor, target_layer):
    gradients = []
    activations = []
    
    def backward_hook(module, grad_input, grad_output):
        gradients.append(grad_output[0])
    
    def forward_hook(module, input, output):
        activations.append(output)
    
    target_layer.register_forward_hook(forward_hook)
    target_layer.register_full_backward_hook(backward_hook)
    
    output = model(input_tensor)
    model.zero_grad()
    
    class_score = output[0, target_class].backward()
    
    gradient = gradients[0].cpu().data.numpy()[0]
    activation = activations[0].cpu().data.numpy()[0]
    
    weights = np.mean(gradient, axis=(1, 2))
    cam = np.zeros(activation.shape[1:], dtype=np.float32)
    
    for i, w in enumerate(weights):
        cam += w * activation[i]
    
    cam = np.maximum(cam, 0)
    cam = cv2.resize(cam, (224, 224))
    cam = cam - np.min(cam)
    cam = cam / np.max(cam)
    
    return cam
```
