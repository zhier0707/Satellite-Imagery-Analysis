## 1. 项目价值反思：用户为什么用这个项目？

### 1.1 灵魂拷问

**EuroSAT 10 类分类 + Grad-CAM + 高德地图，市面上随便一个 HuggingFace Space 都能做。用户凭什么用我们这个项目？**

回答这个问题之前，必须先看清楚现状：当前项目本质上是一个"技术演示"——它证明了团队能搭全栈，但没有解决任何真实问题。

### 1.2 用户使用卫星图像分析平台的三个核心诉求

| 诉求层级 | 问题描述 | 当前项目满足程度 |
|----------|----------|-----------------|
| **基础认知** | 这块地上有什么？（分类/检测） | 浅层满足：仅 10 类、仅 EuroSAT 64×64 小图 |
| **变化感知** | 这块地发生了什么变化？（变化检测） | 几乎未满足：只比标签，不做像素级检测 |
| **决策支持** | 这些信息告诉我什么决策？（分析/洞察） | 完全未满足：只有数据导出，没有智能分析 |

### 1.3 项目的真正价值应该是什么

> **让非遥感专业人员上传一张卫星图，30 秒内得到可用于报告的结构化分析结果。**

这意味着平台必须从"分类器"升级为"分析工具"——分类只是入口，分析才是价值。用户要的不是"这张图 87% 是 Forest"，而是：

- "这片区域 42% 是森林，18% 是河流，25% 是住宅区"
- "相比去年，森林面积减少了 3.2 公顷，住宅区扩张了 1.8 公顷"
- "森林持续减少趋势已持续 3 个季度，建议核查是否存在违法砍伐"

### 1.4 当前项目的核心矛盾

| 矛盾点 | 现状 | 用户期望 |
|--------|------|---------|
| 图像尺寸 | 只认 EuroSAT 64×64 小图 | 真实卫星图（5120×5120 或更大） |
| 分类粒度 | 一张图一个标签 | 一张图一张分类掩码（每像素一个类别） |
| 变化检测 | 标签对比（Forest→River?） | 像素级变化掩码 + 面积统计 |
| 地理坐标 | 硬编码假坐标（Forest→广州白云机场） | 真实 EXIF/用户标注坐标 |
| 数据持久化 | 统计用内存 deque，重启即丢 | 全部基于数据库 |
| 分析深度 | 原始数据搬运 | 智能洞察 + 趋势 + 建议 |

## 2. 现有功能盘点与问题诊断

### 2.1 功能盘点表

| 模块 | 现有功能 | 实现文件 | 核心问题 |
|------|---------|---------|---------|
| **图像分类** | EfficientNetV2-M 对 EuroSAT 10 类做 Top-5 | `backend/api/classify.py`, `backend/services/model_service.py` | 只有 10 类、只认 EuroSAT 的 64×64 小图，换真实卫星图 resize 到 384 会丢大量空间信息 |
| **Grad-CAM** | 类别激活热力图 | `backend/api/classify.py` (heatmap), `src/gradcam.py` | 只是"模型看了哪里"，没有语义解释，无法批量分析 |
| **时相变化** | 两张图 Top-5 标签对比 | `backend/api/change.py` | 只是分类标签对比，两张图都是 Forest 但砍了一半树完全检测不到 |
| **统计** | 7 天分类计数（内存 deque） | `backend/api/stats.py` | 重启即丢，与数据库 ClassifyRecord 完全脱节，不支持多维筛选 |
| **报表** | PDF/Excel/CSV 导出 | `backend/api/reports.py` | 只是原始数据搬运，没有分析洞察 |
| **高德地图** | 分类点热力图 + POI + 地理编码 | `backend/api/lbs.py`, `frontend/src/views/user/MapView.vue` | 分类坐标是硬编码的假坐标（CLASS_COORDS），与图像实际地理位置毫无关系 |
| **训练管理** | Kaggle 脚本 + admin 面板 | `src/train.py`, `backend/api/admin/training.py` | 训练在远端跑，结果无法自动回传，只支持单一模型 |
| **用户系统** | JWT + RBAC（user/admin） | `backend/api/auth.py`, `backend/security/` | 做得最完整的部分，基本无需改动 |
| **前端** | Vue 3 + Element Plus + ECharts + GSAP | `frontend/src/views/` | 视觉完成度高，但交互逻辑基于"单图分类"假设，需要适配大图分析 |

### 2.2 架构问题诊断

#### 2.2.1 数据流断裂

```
当前数据流（断裂的）：
┌─────────┐     ┌──────────┐     ┌──────────┐     ┌──────────┐
│ 上传图像 │ ──→ │ 分类推理  │ ──→ │ 落库记录  │ ──→ │ 地图展示  │
└─────────┘     └──────────┘     └──────────┘     └──────────┘
                      │                                  ↑
                      └──→ 统计（内存）  ──×──→ 不关联 ──┘
                      └──→ 变化检测 ──×──→ 不落库 ──┘
                      └──→ 报表 ──×──→ 只导原始数据 ──┘
```

问题：分类、统计、变化、地图、报表各自为政，数据没有形成闭环。

#### 2.2.2 模型能力瓶颈

- **输入限制**：模型输入 384×384，EuroSAT 原图 64×64（上采样）。用户上传 5120×5120 真实卫星图直接 resize 到 384，空间信息几乎全丢。
- **输出限制**：只有 Top-5 标签，没有空间分布信息（哪块区域是什么类别）。
- **模型单一**：只支持 EfficientNetV2-M，无法切换/对比其他模型。

#### 2.2.3 地理信息虚假

```python
# MapView.vue 中的硬编码假坐标
CLASS_COORDS = {
  "Forest": [113.3, 23.4],    # 广州白云机场附近
  "River": [120.2, 30.3],     # 杭州
  # ...
}
```

用户上传一张北京海淀的卫星图，分类结果为 Forest，地图上却显示在广州——这直接破坏了地图视图的可信度。

#### 2.2.4 统计不可信

```python
# stats.py 用内存 deque 维护统计
_RECORDS: Deque[dict] = deque(maxlen=604800)  # 最多 604800 条
```

- 服务重启 → 统计清零
- 与 ClassifyRecord 表完全脱节
- 不支持按区域、时间、用户等维度筛选
- 不支持环比/同比/趋势分析

## 4. L1 核心能力补全（P0-P1）

### 4.1 改进项 1：真实卫星图支持——突破 64×64 限制

#### 4.1.1 问题分析

EuroSAT 是 64×64 的小图，模型输入 384×384（上采样）。用户上传真实卫星图（如 Sentinel-2 的 10000×10000）时，直接 resize 到 384 会丢失大量空间信息。一张 5120×5120 的图 resize 到 384×384，相当于把 2844 个像素压缩成 1 个，所有空间结构全部消失。

#### 4.1.2 改进方案：滑窗分类（Tile Classification）

```
用户上传大图 (如 5120×5120)
    │
    ▼ 滑窗切片 (tile_size=384, stride=320, overlap=64)
┌──────┬──────┬──────┬──────┬──────┐
│Patch1│Patch2│Patch3│Patch4│Patch5│
├──────┼──────┼──────┼──────┼──────┤
│Patch6│Patch7│Patch8│Patch9│...   │
└──────┴──────┴──────┴──────┴──────┘
    │
    ▼ 每块独立分类（批量推理 batch_size=8）
{Forest:0.9} {River:0.7} {Residential:0.8} ...
    │
    ▼ 拼接回原图空间
分类掩码图 (H×W 矩阵，每像素一个类别索引)
    │
    ▼ 聚合统计
Forest: 42% | River: 18% | Residential: 25% | ...
    │
    ▼ 可选：对每块做 Grad-CAM
Grad-CAM 拼接图（可解释性）
```

#### 4.1.3 具体实现

**新增文件**：`backend/services/tile_classifier.py`

```python
"""
大图滑窗分类器
=============
将大图切分为重叠小块，每块独立分类，再拼接回原图空间。
"""
from __future__ import annotations

import logging
import math
from typing import Any, TYPE_CHECKING

import numpy as np
from PIL import Image

if TYPE_CHECKING:
    import torch
    from sqlalchemy.orm import Session

from backend.classes import EUROSAT_CLASSES

log = logging.getLogger(__name__)


class TileClassifier:
    """大图滑窗分类器"""
    
    def __init__(
        self,
        model_service: Any,
        tile_size: int = 384,
        overlap: int = 64,
        batch_size: int = 8,
    ) -> None:
        self._model_service = model_service
        self.tile_size = tile_size
        self.overlap = overlap
        self.stride = tile_size - overlap
        self.batch_size = batch_size
    
    def classify_large_image(self, img: Image.Image) -> dict:
        """对大图做滑窗分类。
        
        Returns:
            {
                "classification_map": np.ndarray (H×W),  # 每像素类别索引
                "coverage": {label: percentage},
                "top1": str,  # 面积最大的类别
                "tile_count": int,
                "tiles": [...],  # 每块的 top5（供 Grad-CAM 使用）
                "image_size": (W, H),
                "mock": bool,
            }
        """
        w, h = img.size
        tiles_x = math.ceil((w - self.overlap) / self.stride)
        tiles_y = math.ceil((h - self.overlap) / self.stride)
        
        # 1. 切片
        tiles = []
        coords = []
        for ty in range(tiles_y):
            for tx in range(tiles_x):
                x0 = min(tx * self.stride, w - self.tile_size)
                y0 = min(ty * self.stride, h - self.tile_size)
                tile = img.crop((x0, y0, x0 + self.tile_size, y0 + self.tile_size))
                tiles.append(tile)
                coords.append((x0, y0))
        
        # 2. 批量推理
        all_results = []
        for i in range(0, len(tiles), self.batch_size):
            batch = tiles[i:i + self.batch_size]
            batch_results = self._batch_predict(batch)
            all_results.extend(batch_results)
        
        # 3. 拼接分类掩码
        cls_map = np.zeros((h, w), dtype=np.int32)
        for (x0, y0), result in zip(coords, all_results):
            top1_label = result["top1"]["label"]
            cls_idx = EUROSAT_CLASSES.index(top1_label)
            cls_map[y0:y0 + self.tile_size, x0:x0 + self.tile_size] = cls_idx
        
        # 4. 聚合覆盖率
        coverage = self._compute_coverage(cls_map)
        
        return {
            "classification_map": cls_map,
            "coverage": coverage,
            "top1": max(coverage, key=coverage.get),
            "tile_count": len(tiles),
            "tiles": all_results,
            "image_size": (w, h),
            "mock": all_results[0].get("mock", True) if all_results else True,
        }
    
    def _batch_predict(self, tiles: list) -> list:
        """批量推理一组 tile"""
        results = []
        for tile in tiles:
            result = self._model_service.predict(tile)
            results.append(result)
        return results
    
    @staticmethod
    def _compute_coverage(cls_map: np.ndarray) -> dict:
        """计算各类别覆盖率"""
        total = cls_map.size
        coverage = {}
        for idx, label in enumerate(EUROSAT_CLASSES):
            count = int((cls_map == idx).sum())
            if count > 0:
                coverage[label] = round(count / total, 4)
        return dict(sorted(coverage.items(), key=lambda x: -x[1]))
```

### 4.2 改进项 2：像素级变化检测——替代标签对比

#### 4.2.1 问题分析

当前 `/api/change` 只对比两张图的 Top-5 分类标签。如果两张图都是 Forest，但一张砍了一半树，当前系统完全检测不到。真正的变化检测应该做到像素级。

#### 4.2.2 改进方案

```
图A ──┐
       ├── 像素级差异计算
图B ──┘
         │
         ▼
    ┌────┴────┐
    │         │
  分类掩码    图像差异
  变化对比    (NDVI差值 / SSIM / 结构差异)
    │         │
    └────┬────┘
         ▼
  变化掩码 (H×W 二值图：变化/未变化)
         │
         ▼
  变化统计：
  - Forest → Bare Soil: 15.3 公顷
  - 新增建筑: 2.1 公顷
  - 水体减少: 8.7 公顷
  - 总变化面积: 26.1 公顷 (12.3%)
```

#### 4.2.3 具体实现

**新增文件**：`backend/services/change_detection_service.py`

```python
"""
像素级变化检测服务
================
融合分类掩码变化和像素差异，输出变化掩码和面积统计。
"""
from __future__ import annotations

import logging
from typing import Any, TYPE_CHECKING

import cv2
import numpy as np
from PIL import Image

if TYPE_CHECKING:
    pass

from backend.classes import EUROSAT_CLASSES

log = logging.getLogger(__name__)


class ChangeDetectionService:
    """像素级变化检测"""
    
    def __init__(self, tile_classifier: Any) -> None:
        self._tile_classifier = tile_classifier
    
    def detect(
        self,
        img_a: Image.Image,
        img_b: Image.Image,
        method: str = "hybrid",
        pixel_threshold: float = 0.5,
    ) -> dict:
        """像素级变化检测
        
        Args:
            img_a: 时相 A 图像
            img_b: 时相 B 图像
            method: 检测方法
                - 'pixel': 纯像素差值（快但噪声大）
                - 'ssim': 结构相似性（抗噪声）
                - 'hybrid': 分类掩码 + 像素差异（推荐）
            pixel_threshold: 像素差异阈值 (0-1)
        
        Returns:
            {
                "change_mask": np.ndarray (H×W bool),
                "change_ratio": float,
                "transitions": [{from, to, area_pct}],
                "summary": str,
                "cls_map_a": np.ndarray,
                "cls_map_b": np.ndarray,
            }
        """
        # 确保两图尺寸一致
        if img_a.size != img_b.size:
            img_b = img_b.resize(img_a.size, Image.LANCZOS)
        
        if method == "hybrid":
            return self._hybrid_detect(img_a, img_b, pixel_threshold)
        elif method == "ssim":
            return self._ssim_detect(img_a, img_b)
        else:
            return self._pixel_detect(img_a, img_b, pixel_threshold)
    
    def _hybrid_detect(self, img_a, img_b, threshold) -> dict:
        """融合分类变化和像素差异"""
        # 1. 两图分别做滑窗分类
        result_a = self._tile_classifier.classify_large_image(img_a)
        result_b = self._tile_classifier.classify_large_image(img_b)
        cls_a = result_a["classification_map"]
        cls_b = result_b["classification_map"]
        
        # 2. 分类变化掩码
        cls_change = (cls_a != cls_b)
        
        # 3. 像素差异（对齐尺寸后）
        gray_a = cv2.cvtColor(np.array(img_a), cv2.COLOR_RGB2GRAY)
        gray_b = cv2.cvtColor(np.array(img_b), cv2.COLOR_RGB2GRAY)
        pixel_diff = cv2.absdiff(gray_a, gray_b)
        
        # Otsu 自动阈值
        _, pixel_change = cv2.threshold(
            pixel_diff, 0, 255, cv2.THRESH_BINARY + cv2.THRESH_OTSU
        )
        pixel_change = pixel_change > 0
        
        # 4. 融合：分类变化 OR 像素变化
        change_mask = cls_change | pixel_change
        
        # 5. 形态学去噪
        kernel = np.ones((5, 5), np.uint8)
        change_mask = cv2.morphologyEx(
            change_mask.astype(np.uint8), cv2.MORPH_OPEN, kernel
        ).astype(bool)
        
        # 6. 变化统计
        transitions = self._compute_transitions(cls_a, cls_b, change_mask)
        summary = self._make_summary(transitions, change_mask)
        
        return {
            "change_mask": change_mask,
            "change_ratio": float(change_mask.mean()),
            "transitions": transitions,
            "summary": summary,
            "cls_map_a": cls_a,
            "cls_map_b": cls_b,
            "coverage_a": result_a["coverage"],
            "coverage_b": result_b["coverage"],
        }
    
    def _ssim_detect(self, img_a, img_b) -> dict:
        """基于 SSIM 的变化检测"""
        gray_a = cv2.cvtColor(np.array(img_a), cv2.COLOR_RGB2GRAY)
        gray_b = cv2.cvtColor(np.array(img_b), cv2.COLOR_RGB2GRAY)
        
        # 计算 SSIM
        from skimage.metrics import structural_similarity
        score, diff = structural_similarity(gray_a, gray_b, full=True)
        diff = (1 - diff) * 255  # 差异图
        _, change_mask = cv2.threshold(
            diff.astype(np.uint8), 0, 255, cv2.THRESH_BINARY + cv2.THRESH_OTSU
        )
        change_mask = change_mask > 0
        
        return {
            "change_mask": change_mask,
            "change_ratio": float(change_mask.mean()),
            "ssim_score": float(score),
            "transitions": [],
            "summary": f"SSIM score: {score:.3f}, change ratio: {change_mask.mean():.1%}",
        }
    
    def _pixel_detect(self, img_a, img_b, threshold) -> dict:
        """纯像素差值"""
        gray_a = cv2.cvtColor(np.array(img_a), cv2.COLOR_RGB2GRAY)
        gray_b = cv2.cvtColor(np.array(img_b), cv2.COLOR_RGB2GRAY)
        diff = cv2.absdiff(gray_a, gray_b)
        change_mask = (diff / 255.0) > threshold
        
        return {
            "change_mask": change_mask,
            "change_ratio": float(change_mask.mean()),
            "transitions": [],
            "summary": f"Pixel diff change ratio: {change_mask.mean():.1%}",
        }
    
    @staticmethod
    def _compute_transitions(cls_a, cls_b, change_mask) -> list:
        """计算类别转移矩阵"""
        transitions = []
        for from_idx, from_label in enumerate(EUROSAT_CLASSES):
            for to_idx, to_label in enumerate(EUROSAT_CLASSES):
                if from_idx == to_idx:
                    continue
                mask = (cls_a == from_idx) & (cls_b == to_idx) & change_mask
                count = int(mask.sum())
                if count > 0:
                    transitions.append({
                        "from": from_label,
                        "to": to_label,
                        "pixel_count": count,
                        "area_pct": round(count / cls_a.size, 4),
                    })
        transitions.sort(key=lambda x: -x["pixel_count"])
        return transitions
    
    @staticmethod
    def _make_summary(transitions, change_mask) -> str:
        """生成自然语言摘要"""
        ratio = change_mask.mean()
        if ratio < 0.02:
            base = "两期图像变化极小"
        elif ratio < 0.10:
            base = f"两期图像存在少量变化（{ratio:.1%}）"
        elif ratio < 0.30:
            base = f"两期图像存在明显变化（{ratio:.1%}）"
        else:
            base = f"两期图像存在剧烈变化（{ratio:.1%}）"
        
        if transitions:
            top3 = transitions[:3]
            parts = [f"{t['from']}→{t['to']}({t['area_pct']:.1%})" for t in top3]
            base += "，主要变化：" + "、".join(parts)
        
        return base
```

```python
import json
import logging
from typing import Optional

log = logging.getLogger(__name__)


class GeoService:
    """图像地理坐标提取"""
    
    @staticmethod
    def extract_from_exif(img_bytes: bytes) -> Optional[dict]:
        """从 EXIF 提取 GPS 坐标"""
        try:
            from PIL import Image
            from PIL.ExifTags import GPSTAGS, TAGS
            img = Image.open(io.BytesIO(img_bytes))
            exif_data = img._getexif()
            if not exif_data:
                return None
            
            gps_info = {}
            for tag_id, value in exif_data.items():
                tag = TAGS.get(tag_id, tag_id)
                if tag == "GPSInfo":
                    for gps_tag_id, gps_value in value.items():
                        gps_tag = GPSTAGS.get(gps_tag_id, gps_tag_id)
                        gps_info[gps_tag] = gps_value
            
            if "GPSLatitude" in gps_info and "GPSLongitude" in gps_info:
                lat = GeoService._dms_to_decimal(gps_info["GPSLatitude"])
                lng = GeoService._dms_to_decimal(gps_info["GPSLongitude"])
                if gps_info.get("GPSLatitudeRef") == "S":
                    lat = -lat
                if gps_info.get("GPSLongitudeRef") == "W":
                    lng = -lng
                return {"lng": lng, "lat": lat, "source": "exif"}
        except Exception as e:
            log.debug("EXIF GPS extraction failed: %s", e)
        return None
    
    @staticmethod
    def extract_from_geotiff(img_bytes: bytes) -> Optional[dict]:
        """从 GeoTIFF 提取地理边界"""
        try:
            import rasterio
            with rasterio.MemoryFile(img_bytes) as memfile:
                with memfile.open() as dataset:
                    bounds = dataset.bounds
                    crs = dataset.crs
                    # 如果不是 WGS84，需要转换
                    if crs and crs.to_epsg() != 4326:
                        from rasterio.warp import transform_bounds
                        bounds = transform_bounds(crs, "EPSG:4326",
                                                   bounds.left, bounds.bottom,
                                                   bounds.right, bounds.top)
                    lng = (bounds[0] + bounds[2]) / 2
                    lat = (bounds[1] + bounds[3]) / 2
                    return {
                        "lng": lng,
                        "lat": lat,
                        "bbox": [bounds[0], bounds[1], bounds[2], bounds[3]],
                        "source": "geotiff",
                    }
        except ImportError:
            log.debug("rasterio not installed, skipping GeoTIFF")
        except Exception as e:
            log.debug("GeoTIFF extraction failed: %s", e)
        return None
    
    @staticmethod
    def _dms_to_decimal(dms) -> float:
        """度分秒转十进制"""
        degrees, minutes, seconds = dms
        return float(degrees) + float(minutes) / 60 + float(seconds) / 3600
```

#### 5.1.1 为什么需要

分类只告诉你"这张图主要是 Forest"，分割告诉你"Forest 在哪里、占了多少、边界在哪里"。这是从"标签"到"地图"的质变。

#### 5.1.2 方案

| 方面 | 选择 |
|------|------|
| 模型 | DeepLabV3+ MobileNetV2（轻量）/ Mask2Former Swin-T（精度）/ SAM（零样本） |
| 数据集 | LoveDA (7 类) / DeepGlobe (6 类) |
| 接口 | `POST /api/segment` |
| 输入 | 一张卫星图 |
| 输出 | 分割掩码(PNG) + 各类别面积占比 + 各类别边界多边形(GeoJSON) |

**新增文件**：`backend/services/segmentation_service.py`

```python
"""语义分割服务"""
from __future__ import annotations

import logging
from typing import Any
import numpy as np
from PIL import Image

log = logging.getLogger(__name__)


class SegmentationService:
    """地物语义分割"""
    
    SEG_CLASSES = [
        "building", "road", "water", "barren", "forest",
        "agriculture", "background",
    ]
    
    def __init__(self, model_manager: Any) -> None:
        self._model_manager = model_manager
    
    def segment(self, img: Image.Image) -> dict:
        """语义分割
        
        Returns:
            {
                "segmentation_map": np.ndarray (H×W),
                "coverage": {class: percentage},
                "polygons": [{class, geojson}],
                "visualization": Image.Image,
            }
        """
        # 加载分割模型（与分类模型独立）
        model = self._model_manager.get_model("segmenter")
        if model is None:
            return self._mock_segment(img)
        
        # 推理
        import torch
        transform = self._get_transform()
        x = transform(img).unsqueeze(0)
        if torch.cuda.is_available():
            x = x.cuda()
        
        with torch.no_grad():
            out = model(x)
            if isinstance(out, dict):
                out = out["out"]
            pred = out.argmax(1).squeeze(0).cpu().numpy()
        
        # 统计
        coverage = {}
        for idx, label in enumerate(self.SEG_CLASSES):
            pct = float((pred == idx).mean())
            if pct > 0.001:
                coverage[label] = round(pct, 4)
        
        # 提取多边形
        polygons = self._extract_polygons(pred)
        
        return {
            "segmentation_map": pred,
            "coverage": dict(sorted(coverage.items(), key=lambda x: -x[1])),
            "polygons": polygons,
            "image_size": img.size,
        }
    
    @staticmethod
    def _extract_polygons(seg_map: np.ndarray) -> list:
        """从分割图提取类别边界多边形"""
        import cv2
        polygons = []
        for idx, label in enumerate(SegmentationService.SEG_CLASSES):
            mask = (seg_map == idx).astype(np.uint8)
            if mask.sum() < 100:  # 太小的不提取
                continue
            contours, _ = cv2.findContours(
                mask, cv2.RETR_EXTERNAL, cv2.CHAIN_APPROX_SIMPLE
            )
            for contour in contours:
                if len(contour) < 3:
                    continue
                # 简化多边形
                epsilon = 0.01 * cv2.arcLength(contour, True)
                approx = cv2.approxPolyDP(contour, epsilon, True)
                if len(approx) < 3:
                    continue
                polygons.append({
                    "class": label,
                    "points": approx.squeeze().tolist(),
                })
        return polygons
    
    def _mock_segment(self, img: Image.Image) -> dict:
        """无模型时的 mock"""
        w, h = img.size
        seg_map = np.zeros((h, w), dtype=np.int32)
        # 简单分块
        seg_map[:h//2, :w//2] = 4  # forest
        seg_map[:h//2, w//2:] = 2  # water
        seg_map[h//2:, :w//2] = 0  # building
        seg_map[h//2:, w//2:] = 4  # agriculture
        return {
            "segmentation_map": seg_map,
            "coverage": {"forest": 0.25, "water": 0.25, "building": 0.25, "agriculture": 0.25},
            "polygons": [],
            "image_size": (w, h),
            "mock": True,
        }
```

```python
# backend/services/trend_analysis_service.py (新增)
import numpy as np
from collections import defaultdict
from datetime import datetime


class TrendAnalysisService:
    """多时序趋势分析"""
    
    def analyze(self, records: list) -> dict:
        """
        Args:
            records: 按时间排序的 ClassifyRecord 列表
        """
        # 构建时间线
        timeline = []
        for r in records:
            import json
            coverage = json.loads(r.coverage_json) if r.coverage_json else {}
            timeline.append({
                "date": r.created_at.strftime("%Y-%m"),
                "coverage": coverage,
                "record_id": r.id,
            })
        
        if len(timeline) < 2:
            return {"timeline": timeline, "trends": [], "anomalies": []}
        
        # 趋势分析（线性回归）
        trends = []
        all_labels = set()
        for t in timeline:
            all_labels.update(t["coverage"].keys())
        
        for label in all_labels:
            values = [t["coverage"].get(label, 0) for t in timeline]
            x = np.arange(len(values))
            y = np.array(values)
            
            if len(values) >= 3:
                slope, intercept = np.polyfit(x, y, 1)
                r2 = self._r_squared(x, y, slope, intercept)
                narrative = self._narrative(label, slope, r2)
                trends.append({
                    "label": label,
                    "slope": round(float(slope), 6),
                    "r2": round(float(r2), 4),
                    "current": values[-1],
                    "start": values[0],
                    "change": round(values[-1] - values[0], 4),
                    "narrative": narrative,
                })
        
        trends.sort(key=lambda t: abs(t["change"]), reverse=True)
        
        # 异常检测
        anomalies = self._detect_anomalies(timeline)
        
        return {
            "timeline": timeline,
            "trends": trends,
            "anomalies": anomalies,
        }
    
    @staticmethod
    def _r_squared(x, y, slope, intercept):
        y_pred = slope * x + intercept
        ss_res = np.sum((y - y_pred) ** 2)
        ss_tot = np.sum((y - np.mean(y)) ** 2)
        return 1 - ss_res / ss_tot if ss_tot > 0 else 0
    
    @staticmethod
    def _narrative(label, slope, r2):
        if abs(slope) < 0.005:
            return f"{label} 保持稳定"
        direction = "减少" if slope < 0 else "增加"
        confidence = "显著" if r2 > 0.85 else "缓慢" if r2 > 0.6 else "波动"
        return f"{label} {confidence}{direction}"
    
    @staticmethod
    def _detect_anomalies(timeline):
        """检测突变"""
        anomalies = []
        for i in range(1, len(timeline)):
            for label in set(list(timeline[i]["coverage"].keys()) +
                           list(timeline[i-1]["coverage"].keys())):
                prev = timeline[i-1]["coverage"].get(label, 0)
                curr = timeline[i]["coverage"].get(label, 0)
                change = curr - prev
                if abs(change) > 0.05:  # 5% 突变阈值
                    anomalies.append({
                        "date": timeline[i]["date"],
                        "type": "sudden_drop" if change < 0 else "sudden_rise",
                        "label": label,
                        "magnitude": round(change, 4),
                    })
        return anomalies
```

### 6.1 改进项 9：异步推理队列

#### 6.1.1 问题

当前推理同步阻塞在 FastAPI 请求里。大图滑窗分类可能要 30 秒，用户 HTTP 请求会超时。

#### 6.1.2 方案

```
POST /api/classify/tiled
  → 立即返回 202 + job_id
  → Celery worker 异步推理
  → 前端轮询 GET /api/jobs/{job_id}
  → 完成后返回结果

技术栈：
  - Redis (broker)
  - Celery (worker)
  - 或更轻量：直接用现有 ThreadPoolExecutor + DB 状态机（已有 reports 先例）
```

**轻量方案（推荐先用）**：复用现有 `ThreadPoolExecutor` 模式

```python
# backend/services/job_queue.py (新增)
"""
轻量异步任务队列
===============
复用 ThreadPoolExecutor 模式（与 reports.py 一致），
通过 DB 状态机跟踪任务进度。
"""
from __future__ import annotations

import logging
import time
import traceback
from concurrent.futures import Future, ThreadPoolExecutor
from typing import Any, Callable

from sqlalchemy.orm import Session

log = logging.getLogger(__name__)

# 全局线程池
_executor = ThreadPoolExecutor(max_workers=4, thread_name_prefix="inference")
_futures: dict[int, Future] = {}


def submit_job(
    job_id: int,
    func: Callable,
    *args,
    db_session_factory: Callable[[], Session],
    **kwargs,
) -> int:
    """提交异步推理任务"""
    future = _executor.submit(_wrap_job, job_id, func, db_session_factory, *args, **kwargs)
    _futures[job_id] = future
    return job_id


def _wrap_job(job_id, func, db_session_factory, *args, **kwargs):
    """包装任务：更新 DB 状态"""
    from backend.db.models import InferenceJob
    try:
        with db_session_factory() as db:
            job = db.query(InferenceJob).get(job_id)
            job.status = "running"
            job.started_at = datetime.utcnow()
            db.commit()
        
        result = func(*args, **kwargs)
        
        with db_session_factory() as db:
            job = db.query(InferenceJob).get(job_id)
            job.status = "completed"
            job.finished_at = datetime.utcnow()
            job.result_json = json.dumps(result, ensure_ascii=False)
            db.commit()
        
        return result
    except Exception as e:
        log.exception("job %s failed: %s", job_id, e)
        with db_session_factory() as db:
            job = db.query(InferenceJob).get(job_id)
            job.status = "failed"
            job.error = str(e)
            job.finished_at = datetime.utcnow()
            db.commit()
        raise


def get_job_status(job_id: int) -> str | None:
    future = _futures.get(job_id)
    if future is None:
        return None
    if future.done():
        return "completed" if not future.exception() else "failed"
    return "running"
```

```python
        self._models: dict[str, Any] = {}
        self._current: str = "efficientnetv2_m"
    
    def get_model(self, name: str = None) -> Any:
        """获取指定模型（默认当前活跃模型）"""
        key = name or self._current
        if key not in self._models:
            self._load_model(key)
        return self._models.get(key)
    
    def switch_model(self, name: str) -> None:
        """切换活跃模型"""
        if name not in self.MODELS:
            raise ValueError(f"Unknown model: {name}")
        self._current = name
    
    def list_models(self) -> dict:
        """列出可用模型"""
        return {
            "current": self._current,
            "available": list(self.MODELS.keys()),
            "loaded": list(self._models.keys()),
        }
```

## 14. 风险评估与缓解措施

| 风险 | 等级 | 缓解措施 |
|------|------|---------|
| 滑窗分类内存溢出（超大图） | 高 | 限制最大图像尺寸（如 10240×10240），分批加载，使用内存映射 |
| 模型推理速度慢 | 中 | 异步队列 + 进度条 + GPU 加速（如果有） |
| GeoTIFF 依赖安装复杂 | 中 | 提供 Docker 镜像预装依赖；非 GeoTIFF 场景不需要 rasterio |
| 数据库迁移破坏现有数据 | 高 | 迁移前备份；新增字段全部 nullable；提供回滚脚本 |
| 前端大图渲染卡顿 | 中 | 使用 Canvas 渐进式渲染；分类掩码预生成 PNG |
| 高德地图配额超限 | 低 | share 接口已有 admin 限制；普通接口走后端代理缓存 |
| 多模型切换时权重文件过大 | 中 | 使用 git-lfs；按需加载；提供模型下载脚本 |
